## Descripción del Cambio

Una descripción clara y concisa de los cambios realizados. Incluye el contexto y la razón de este cambio.

## Tipo de Cambio

Por favor, marca la opción que aplique:

- [ ] Bug fix (cambio que soluciona un problema)
- [ ] New feature (cambio que añade una funcionalidad)
- [ ] Breaking change (cambio que rompe la compatibilidad con versiones anteriores)
- [ ] Refactor (cambio que no añade funcionalidad ni corrige un bug)
